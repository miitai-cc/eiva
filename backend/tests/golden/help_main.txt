Eiva — a super-lightweight super-capable agentic tool with improved security.

Run without a subcommand to launch the TUI.
Use `eiva desktop` for the desktop client.

Usage: eiva [OPTIONS] [COMMAND]

Commands:
  setup          Initialise config + workspace (runs the wizard when wizard flags are present)
  onboard        Interactive onboarding wizard — set up gateway, workspace, and skills
  import         Import an existing OpenClaw installation into Eiva
  configure      Interactive configuration wizard (models, gateway, skills)
  config         Config helpers: get / set / unset
  doctor         Health checks + quick fixes for gateway and configuration
  tui            Launch the terminal UI
    desktop        Launch the desktop UI
  command        Send a one-shot message / slash-command
    ask            Send a prompt to the model and print the response (headless mode)
  status         Show system status (gateway, model, workspace)
  gateway        Gateway management (start / stop / restart / status)
  skills         List / manage skills
  refresh-token  Refresh the GitHub Copilot session token from OpenClaw
  clawhub        ClawHub skill registry commands (search, install, publish, …)
  help           Print this message or the help of the given subcommand(s)

Options:
  -c, --config <PATH>
          Path to a config.toml file

          [env: RUSTYCLAW_CONFIG=]

      --settings-dir <DIR>
          Settings directory (default: ~/.eiva)

          [env: RUSTYCLAW_SETTINGS_DIR=]

      --profile <PROFILE>
          Isolate state under ~/.eiva-<PROFILE>

          [env: RUSTYCLAW_PROFILE=]

      --no-color
          Disable coloured terminal output

          [env: NO_COLOR=]

      --soul <PATH>
          Path to SOUL.md

          [env: RUSTYCLAW_SOUL=]

      --skills <DIR>
          Skills directory

          [env: RUSTYCLAW_SKILLS=]

      --no-secrets
          Disable secrets storage

      --gateway <WS_URL>
          Gateway WebSocket URL (ws://…)

          [env: RUSTYCLAW_GATEWAY=]

  -h, --help
          Print help (see a summary with '-h')

  -V, --version
          Print version
